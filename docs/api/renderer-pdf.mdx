The `@solez-ai/vell-renderer-pdf` package converts a parsed Vell AST document directly into a PDF binary using [pdfkit](https://pdfkit.org/). It walks the AST and emits PDF drawing primitives — it does not go through HTML as an intermediate step. The output is a standard A4 PDF with support for headings, paragraphs, tables, code blocks, math blocks, definition lists, footnotes, cross-references, and theorem environments.

<Warning>
  This package depends on `pdfkit` and uses Node.js `Buffer` APIs. It is
  **not suitable for browser-only environments**. If you need client-side PDF
  generation, render to HTML with `@solez-ai/vell-renderer-html` and use a
  browser print dialog or a headless PDF tool instead.
</Warning>

## Installation

```bash
npm install @solez-ai/vell-renderer-pdf @solez-ai/vell
```

## Functions

### `renderToPdfBuffer()`

Renders a Vell document AST to a PDF binary and returns it as a `Uint8Array`. The function is synchronous.

```typescript
function renderToPdfBuffer(doc: VellDocument): Uint8Array
```

<ParamField body="doc" type="VellDocument" required>
  The parsed Vell document AST, as returned by `parse()` or `parseOrThrow()`
  from `@solez-ai/vell`. The document must not be `null` — validate or
  check for errors before calling this function.
</ParamField>

**Returns:** `Uint8Array` — the raw PDF binary. You can write this directly to a file or send it as an HTTP response with `Content-Type: application/pdf`.

```typescript
import { parse } from '@solez-ai/vell';
import { renderToPdfBuffer } from '@solez-ai/vell-renderer-pdf';
import { writeFileSync } from 'fs';

const source = `= My Report

Author: Jane Smith

== Introduction

This document was generated by Vell.
`;

const result = parse(source);

if (!result.document) {
  console.error('Parse failed:', result.errors[0]?.message);
  process.exit(1);
}

const pdf = renderToPdfBuffer(result.document);
writeFileSync('report.pdf', Buffer.from(pdf));
console.log('PDF written to report.pdf');
```

## Node.js usage patterns

<CodeGroup>
```typescript Write to file
import { parseOrThrow } from '@solez-ai/vell';
import { renderToPdfBuffer } from '@solez-ai/vell-renderer-pdf';
import { writeFileSync } from 'fs';

const doc = parseOrThrow(source);
const pdf = renderToPdfBuffer(doc);
writeFileSync('output.pdf', Buffer.from(pdf));
```

```typescript HTTP response (Express)
import { parseOrThrow } from '@solez-ai/vell';
import { renderToPdfBuffer } from '@solez-ai/vell-renderer-pdf';
import type { Request, Response } from 'express';

function handler(req: Request, res: Response) {
  const doc = parseOrThrow(req.body.source);
  const pdf = renderToPdfBuffer(doc);

  res.setHeader('Content-Type', 'application/pdf');
  res.setHeader('Content-Disposition', 'attachment; filename="document.pdf"');
  res.send(Buffer.from(pdf));
}
```

```typescript Stream to writable
import { parseOrThrow } from '@solez-ai/vell';
import { renderToPdfBuffer } from '@solez-ai/vell-renderer-pdf';
import { createWriteStream } from 'fs';
import { Readable } from 'stream';

const doc = parseOrThrow(source);
const pdf = renderToPdfBuffer(doc);

const readable = Readable.from(Buffer.from(pdf));
readable.pipe(createWriteStream('document.pdf'));
```
</CodeGroup>

## Page layout

The renderer produces A4 pages with the following fixed layout. All measurements are in PDF points (pt), where 1 pt = 1/72 inch.

| Property | Value |
|---|---|
| Page width | 595.28 pt (A4) |
| Page height | 841.89 pt (A4) |
| Margin (all sides) | 56.69 pt (~20 mm) |
| Content width | 481.89 pt |
| Body font | Helvetica, 10 pt |
| Line height | 1.35× font size |

Automatic page breaks are inserted whenever adding the next block would exceed the bottom margin. The renderer tracks a `cursorY` position and calls `doc.addPage()` as needed.

## Node rendering reference

### Block nodes

| AST node | PDF output |
|---|---|
| `Heading` | Bold text at size appropriate to level |
| `Paragraph` | Body text with inline style runs (bold, italic, underline, etc.) |
| `Blockquote` | Indented body text with a left border line |
| `CodeBlock` | Gray background rect, Courier font, optional language label |
| `MathBlock` | Light background rect, Courier font (LaTeX source displayed verbatim) |
| `List` (unordered) | `•` bullet with 14 pt indent |
| `List` (ordered) | `1.` numbering with 14 pt indent |
| `Table` | Manual cell grid with header shading (`#f0f0f0`) and borders (`#ccc`) |
| `HorizontalRule` | Full-width horizontal line in `#999` |
| `DefinitionList` | Bold term followed by indented definition |
| `FootnoteDefinition` | Collected in pre-pass; rendered after a divider at end of document |
| `VarDeclaration` | Not rendered |
| `ForLoop` | Placeholder: `[@for: see interactive viewer]` |
| `IfBlock` | Placeholder: `[@if: see interactive viewer]` |

### Directive nodes

| Directive | PDF output |
|---|---|
| `@[Equation]` | Auto-numbered `(N)` label at right margin, LaTeX source in Courier |
| `@[Theorem]` / `@[Lemma]` / `@[Corollary]` / `@[Definition]` / `@[Axiom]` / `@[Conjecture]` / `@[Proposition]` | Bold label with auto-number, followed by body children |
| `@[Proof]` / `@[Remark]` / `@[Example]` / `@[Notation]` | Bold label (no auto-number), followed by body children |
| `@[Ref]` | Blue underlined display text if label resolves; red italic `[?label]` if unresolved |
| `@[Toc]` | Section heading "Table of Contents" (full entry list requires pre-pass) |
| `@[Lof]` | Section heading "List of Figures" |
| `@[Lot]` | Section heading "List of Tables" |
| `@[Chem]` | Chemical formula in Courier with Unicode subscript digits |
| `@[Diagram]` | Type label `[Diagram: type]`, source in Courier, optional caption |
| `@[Chart]` | Title followed by `label: value` rows in body font |
| `@[Plot]` | Bold function expression, italic note about interactive viewer |
| `@[Meta]` | Not rendered |
| Unknown directives | Muted italic placeholder `[DirectiveName directive]`, children rendered |

### Inline nodes

All inline content is broken into **styled text spans** that are merged into runs before being written to the page. The following style combinations map to pdfkit fonts:

| Style | Font |
|---|---|
| Normal | Helvetica |
| Bold | Helvetica-Bold |
| Italic | Helvetica-Oblique |
| Bold + Italic | Helvetica-BoldOblique |
| Monospace (inline code) | Courier |
| Underline | Helvetica + `{ underline: true }` option |
| Strikethrough | Helvetica + `{ strike: true }` option |

Links include the destination URL in parentheses: `link text (https://example.com)`.  
Images render as `[Image: alt text]` placeholder text.  
Inline math renders as `$latex source$` in Courier.

## Footnotes

Footnotes are handled in two passes:

1. **Pre-scan pass** — Before rendering, the renderer scans all `FootnoteDefinition` nodes in the document and stores their plain-text content in a lookup map keyed by marker.
2. **Render pass** — When a `FootnoteRef` inline node is encountered, its marker is added to the footnote collection list.
3. **End of document** — After all blocks are rendered, if any footnotes were referenced, a section is appended:
   - A horizontal divider line in `#ccc`
   - A bold "Footnotes" heading at 8 pt
   - Each entry as `[marker] body text` at 8 pt Helvetica

```
────────────────────────────────────────
Footnotes
[1] This is the first footnote body.
[2] Another footnote.
```

## Cross-references

The renderer implements a label pre-pass that mirrors the HTML renderer, enabling `@[Ref]` directives to resolve to numbered equations and theorem environments:

- **Equations** are numbered sequentially across the document starting at 1. A `@[Ref](label=my-eq)` pointing to the third equation renders as `(3)`.
- **Theorems, Lemmas, Corollaries, Definitions, Axioms, Conjectures, and Propositions** are each numbered independently within their type. `Proof`, `Remark`, `Example`, and `Notation` are not auto-numbered.
- **Resolved references** render as blue (`#2b6cb0`) underlined text.
- **Unresolved references** render as red italic `[?label]`.
